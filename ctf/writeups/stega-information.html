Image Metadata — Base64 in EXIF

Challenge débutant : fouiller les métadonnées puis décoder un indice.
Catégorie : Steganalysis Niveau : Beginner Outils : exiftool • base64 Technique : metadata inspection

Contexte

On reçoit une image (ex: cat.jpg ou équivalent). Visuellement elle a l’air normale. L’indice typique : “les fichiers peuvent mentir” → on regarde la structure et les métadonnées.

[IMAGE ICI : l’image du challenge] Ex: une image “banale”

Approche

  • Vérifier le type de fichier (file)
  • Extraire les métadonnées (exiftool)
  • Repérer une chaîne suspecte (souvent Base64)
  • Décoder la chaîne (base64 -d)

Étapes (résolution)

1) Identifier le fichier

file cat.jpg

2) Lire les métadonnées

exiftool cat.jpg

On inspecte surtout les champs : Comment, Description, Artist, Copyright, ou tout champ qui contient une chaîne “longue” (souvent Base64).

[IMAGE ICI : sortie exiftool avec la ligne suspecte] Entoure la ligne “Comment” / “Description”

3) Décoder l’indice (si la chaîne ressemble à Base64)

echo "VFlQRV9JQ0lfQkFTRTY0X0NPTU1FTlQ=" | base64 -d

(Exemple illustratif : toi tu remplaceras la chaîne par celle que tu verras dans tes métadonnées.)

[IMAGE ICI : terminal montrant le décodage Base64] Preuve : commande + résultat

Résultat

Le décodage donne un texte lisible, souvent directement le flag, ou un chemin/indice final. Pour ton portfolio, tu peux afficher : 

FLAG{EXIF_METADATA_BASE64_DECODED}
[IMAGE ICI : preuve finale (flag / message)] Capture du résultat final

Ce que j’ai retenu

  • Les métadonnées sont un réflexe simple et très rentable en CTF.
  • Base64 revient tout le temps : savoir reconnaître et décoder rapidement.
  • Un writeup pro = étapes + preuves (captures + commandes).

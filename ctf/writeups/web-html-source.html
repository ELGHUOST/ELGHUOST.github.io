HTML Source Code — Hidden Password

Challenge débutant : retrouver un mot de passe caché dans le code source.
Catégorie : Web Niveau : Beginner Compétences : DevTools • lecture HTML Livrable : password/flag

Contexte

Une page web affiche un formulaire de “login” très basique. L’énoncé suggère qu’il faut “regarder autrement” la page. Dans ce type de challenge, la solution est souvent cachée dans le code HTML (commentaire, champ hidden, etc.).

Capture de la page du challenge (vue normale)

Approche

  • Ouvrir les DevTools (F12) et inspecter le DOM
  • Afficher le code source de la page
  • Chercher des mots-clés : password, flag, secret, key
DevTools ouvert sur “Elements”

Étapes (résolution)

  • Click droit → Afficher le code source (ou Ctrl+U selon navigateur)
  • Dans la page source, utiliser Ctrl+F et chercher “password” / “flag”
  • Repérer un commentaire HTML ou un input caché qui contient la valeur attendue
  • Revenir sur la page du challenge, entrer la valeur et valider
Exemple typique rencontré dans ce genre de challenge (illustratif) :



ou

<input type="hidden" name="secret" value="[VALEUR_CACHEE]" />
Code source : ligne avec la valeur cachée (surlignée)

Résultat

Une fois la valeur repérée dans le source, il suffit de la soumettre dans le formulaire. Pour ton portfolio, tu peux afficher un résultat du type : 

FLAG{HTML_SOURCE_CODE_FOUND}
Validation / page de succès (preuve finale)

Ce que j’ai retenu

  • Ne jamais se limiter à ce qui est “visible” : inspecter le code et les ressources.
  • DevTools = réflexe de base en Web CTF.
  • Un bon writeup explique la logique, pas juste “où cliquer”.
